开始

创建文件 HelloWorld.java(文件名需与类名一致)

  • 类名：对于所有的类来说，类名的首字母应该大写。如果类名由若干单词组成，那么每个单词的首字母应该大写，例如 MyFirstJavaClass

  • 方法名：所有的方法名都应该以小写字母开头。如果方法名含有若干单词，则后面的每个单词首字母大写算

  • 源文件名：源文件名必须和类名相同。当保存文件的时候，你应该使用类名作为文件名保存（切记 Java 是大小写敏感的），文件名的后缀为 .java。（如果文件名和类名不相同则会导致编译错误）

  • 主方法入口：所有的 Java 程序由 public static void main(String []args) 方法开始执行。

  • 编译单元：一个.java文件是一个编译单元，一个编译单元里可以有很多类，最多只有一个类可以是public

  • 包

    • 在引用别的包里的类是需要用到import，或者选择输出该类的全名
      我们也可以选择import package_name.*而引入该包里全部的类，但不建议这么做，有可能会产生重名冲突

    • 包的名字可以是xxx.xxx，这里不妨将包理解为“文件夹”，是一种管理类的单元

      如java.util.Scanner中的点就表示了文件夹里的层次

  • ALT+/，eclipse可以智能联想补全

  • Shift+方向键可以键盘选定内容

  • 如上选定多行后Ctrl+/可以一次性注释多行

循环

for 循环

在for循环中定义的变量除了for循环就不可用了，这样蛮好的，可以随用随取

for-each 循环

特别适合遍历读出数组中的值

  • 但是注意这样对数组元素的修改写入操作是不能成功的

    这样print出的数组元素还是之前的值，并不是修改后的0

  • 任何返回一个数组的方法都可以用foreach，例如String类有一个方法toCharArray()，返回一个char数组

  • 许多for语句都在整型值序列中步进，如for (i = 0; i < 100; i++)

    对于这些语句，foreach不能直接实现，除非创建一个int数组。

    为了简化这些工作，作者在net.mindview.util.Range包中创建了一个名为range()的方法

循环控制

同c语言一样，break可以跳出当前循环，但如果想要直接跳出多重循环，可以使用标记（像MIPS 的J指令，C语言的goto）

要记住：在Java里要使用标签的唯一理由就是因为有循环嵌套存在

逻辑类型

布尔类型

boolean flag = ture;

boolean flag = faulse;

Java和c不一样在于普通的int类型变量不能进行逻辑运算，不支持!n，其中n是int类型，一定要布尔类型才可以。在逻辑判断中也不能直接认为值不为0的整型为真。也不能强制类型转换(boolean)n;。

数组

数组的元素

  • Java中的定义数组和c语言不太一样，首先[]放置的位置不一样，其次在定义的时候，数组元素数可以是一个变量

  • Java的数组中还含有一个固有成员length因此更推荐的写法应如下（具有可扩展性）

  • 除了用new的方法创建数组，和c语言一样也可以直接给出初值，就不用new了

  • Java的数组类似c语言的指针啊，怪不得一定要先new创建一个数组出来呢，就跟不能用空指针一个道理。数组变量之间的赋值是管理权限的赋予，就好像指针和指针的赋值：两个指针指向同一个变量的话，改变会对两者都产生影响。数组变量之间的比较是判断是否管理同一个数组。

  • 鉴于上一点所述，数组变量之间的赋值只是管理权限的赋予，如果想要复制一个数组，必须采取遍历的操作。

    之后再执行System.out.println(a==b)得到的结果为false

  • 创建数组会默认初始化为零

  • Math.sqrt(x)

字符串

字符串变量和数组变量类似，它并不存放字符串，不是字符串的所有者，它是字符串的管理者。

Java的字符串还是一种特殊的“不可变”对象，所有的字符串操作都是产生一个新的字符串，而不是对原来的字符串的修改。

  • 用加号可以连接两个字符串

  • 比较两个String

字符串是对象，对它的所有操作都是通过.完成的，可以是常量也可以是变量

  • 访问单个字符

  • 得到子串

    s.substring(n)得到从n号到末尾的子串

    s.substring(b,e)得到从b号到e号之前的子串

  • 寻找字符（具体略）

  • 在switch-case中使用字符串

  • replace与replaceAll分别是字符替换和正则表达式替换

类与对象

成员变量与成员函数

  • this

    任何一个成员函数里，总是可以用this来表达这一次调用这个函数的对象

    而且如果成员变量输入太复杂，直接输入this.自动补全也是一个偷懒的好办法

  • 成员变量的生存期和作用域是函数内部

  • 成员变量的生存期是对象的生存期，作用域是类内部的成员函数

    Java会自动回收无用变量

  • Java的成员变量在定义的地方就可以给出初值，没有给出初值的成员变量会自动获得0值

对象初始化

  • 构造函数
    如果有一个成员函数的名字和类的名字完全相同，则在创建这个类的每一个对象是会自动调用这个函数。该函数不能有返回类型

  • 函数重载

    一个类里同名但参数表不同的函数构成了重载关系

    成员函数中可以有多个构造函数，具体调用哪个取决于给的参数类型

抽象与接口

表达概念而无法实现具体代码的函数或类，带有abstract修饰符

  • 有抽象函数的类一定是抽象类

  • 抽象类不能制造对象

  • 但是可以定义变量

    • 任何继承了抽象类的非抽象类的对象可以付给这个变量

  • 继承自抽象类的子类必须覆盖父类中的抽象函数，否则自身也是抽象函数

  • 接口是纯抽象类

    • 所有的成员函数都是抽象函数
    • 所有的成员变量都是public static final

  • 接口规定了长什么样，但不管里面有什么

面向接口的编程方式

  • 设计程序时先定义接口再实现类
  • 任何需要在函数间传入传出的一定是接口而不是具体的类

访问属性

  • private——私有

    只有在这个类的内部才能访问

    • 类内部指类的成员函数和定义初始化
    • 这个限制是对类的不是对对象的。即，一个类内所有对象之间可以访问成员变量

  • public——公有

    类前的public表示任何人都可以用这个类的定义来定义变量，且这个类必须处在和其同名的源代码文件里

  • frendly——同一个包内可以访问

    所有不带private和public的类型都为friendly

  • protect

类变量和类函数

static表明这个类或函数不属于任何一个对象，而是属于这个类

在类函数里也可以调用别的类函数

[对象容器][https://blog.csdn.net/a2011480169/article/details/52047600]

在Java当中，如果有一个类专门用来存放其它类的对象，这个类就叫做容器，或者就叫做集合，集合就是将若干性质相同或相近的类对象组合在一起而形成的一个整体

  • 容器接口（Collection，Map，Iterator）

    • Collection接口

    • Map接口（存放键值对，Map中的值也可以是一个容器）

    • Iterator接口

  • 子接口（List，Set，ListIterator，SortedMap，SortedSet）

    • List（有顺序可以重复，有顺序所以操作时可以在方法中加入索引参数）

    • Set（无顺序不可以重复，无序因而不能通过索引操作对象）

    • ListIterator（Iterator for List，List是双向表，因而在Iterator上增加了一些新的方法，允许traverse the List in either direction）

    • SortedMap(保证按照键的升序排列的映射，可以按照键的自然顺序（ Comparable 接口）进行排序， 或者通过创建有序映射时提供的比较器进行排序)

    • SortedSet(主要用于排序操作，实现此接口的子类都是排序的子类)

  • 抽象容器类

    使用抽象容器类可以方便的定义类，而不用在每个类中都实现容器接口container 中的所有的方法

  • 容器与数组的关系

    • 之所以需要容器：
      1. 数组的长度难以扩充
      2. 数组中数据的类型必须相同
    • 容器与数组的区别与联系：
      1. 容器不是数组，不能通过下标的方式访问容器中的元素
      2. 数组的所有功能通过Arraylist容器都可以实现，只是实现的方式不同
    1. 如果非要将容器当做一个数组来使用，通过toArray方法返回的就是一个数组

  • 容器常用的几个方法

  • 容器的分类
    容器分为Set集、List列表、Map映射

    • Set集合：由于内部存储结构的特点，Set集合中不区分元素的顺序（即使存放的类实现了compareTo方法，也是没用的），不允许出现重复的元素（用户自定义的类有的时候需要实现相应方法），TreeSet容器特殊，元素放进去的时候自然而然就有顺序了，Set容器可以与数学中的集合相对应：相同的元素不会被加入。
    • List列表：由于内部存储结构的特点，List集合中区分元素的顺序，且允许包含重复的元素。List集合中的元素都对应一个整数型的序号记载其在容器中的位置，可以根据序号存取容器中的元素—有序，可以重复
    • Map映射：由于内部存储结构的特点，映射中不能包含重复的键值，每个键最多只能映射一个值，否则会出现覆盖的情况(后面的value值会将前面的value值覆盖掉)，Map是一种把键对象和值对象进行映射的集合，即Map容器中既要存放数据本身，也要存放关键字:相同的元素会被覆盖

  • Comparable接口中的compareTo()方法:

    • 凡是需要进行比较排序的类都应该实现Comparable接口中的compareTo()方法
    • 凡是把类对象放到以树为内部结构的容器中都应该实现Comparable接口中的compareTo()方法

    输出结果：

  • 凡是把类对象放到以哈希表为内部存储结构的容器中，相应的类必须要实现equals方法和hashCode方法，这样才符合哈希表真实的逻辑功能

    • 重写equals()和hashcode()

  • 逻辑上来讲，只要两个对象的内容相同，其地址(hashCode()返回值)以及这两个对象就应该相同(equals())

  • 凡是把类对象放到容器中，相应的类都应该实现Object类中的toString()方法

  • iterator接口的作用

  • 哈希冲突的相关概念

顺序容器

我们首先学习的是顺序容器，即放进容器中的对象是按照指定的顺序（放的顺序）排列起来的，而且允许具有相同值的多个对象存在。

  • 容器类有两个类型

    • 容器的类型——ArrayList
    • 元素的类型——String

  • 泛型容器类

    private ArrayList<String> notes = new ArrayList<String>();

ArrayList

  • 简介

    ArrayList是一个其容量能够动态增长的动态数组。

    基本的ArrayList，长于随机访问元素，但是在List中间插入和移除元素时较慢。同时，ArrayList的操作不是线程安全的！一般在单线程中才使用ArrayList，而在多线程中一般使用Vector或者CopyOnWriteArrayList。

  • 遍历方式

    1. 迭代器遍历

    2. 索引值遍历

    3. for循环遍历

  • 初始化（展示的为源码）

    1. 默认构造函数

    2. 带初始容量参数的构造函数

    3. 带Collection参数的构造函数

  • ArrayList依赖于数组实现的,初始长度为10的Object[],并且可随需要而增加的动态数组当元素超过10,那么ArrayList底层会新生成一个数组,长度为原来的1.5倍+1,然后将原数组内容复制到新数组中,并且后续增加的内容会放到新数组中,当新数组无法容纳增加的元素,重复该过程

  • ArrayList对随机访问性能很好,但进行大量插入,删除操作,性能很差,因为操作之后后续元素需要移动

继承与多态

父类成员访问属性 在父类中的含义 在子类中的含义

protected 只有包内其它类、自己和子类可以访问 只有包内其它类、自己和子类可以访问

正则表达式

  • Pattern类：

    pattern对象是一个正则表达式的编译表示。Pattern类没有公共构造方法。要创建一个Pattern对象，你必须首先调用其公共静态编译方法，它返回一个Pattern对象。该方法接受一个正则表达式作为它的第一个参数。

  • Matcher类：

    Matcher对象是对输入字符串进行解释和匹配操作的引擎。与Pattern类一样，Matcher也没有公共构造方法。你需要调用Pattern对象的matcher方法来获得一个Matcher对象。

  • PatternSyntaxException：

    PatternSyntaxException是一个非强制异常类，它表示一个正则表达式模式中的语法错误。
